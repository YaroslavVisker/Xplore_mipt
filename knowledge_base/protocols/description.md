Purpose: Storage of the screening/eligibility protocol definitions.

Expected contents:

Protocols authored in JSON/YAML (or your custom DSL)

Versioned protocol definitions (e.g., diabetes_v1.json, diabetes_v2.yaml)

Criteria definitions including:
- inclusion/exclusion criteria
- thresholds and cutoffs
- lists of required features
- logical expressions for matching patients

Metadata:
- protocol ID
- title, description
- version history
- last updated timestamp

This folder is the heart of the Knowledge Representation Layer.
